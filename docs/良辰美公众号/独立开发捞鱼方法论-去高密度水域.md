# 独立开发捞鱼方法论：去高密度水域，而不是去竞争者最多的地方

独立开发赚钱的本质，不是找灵感，也不是凭空想 idea，而是寻找“不平衡”。

所谓不平衡，就是：

> 需求已经发生，用户已经在搜索、提问、抱怨、复制粘贴、手动处理，甚至已经愿意付钱，但市场上还没有一个刚好打中他的低成本解决方案。

钱不在产品里，钱在已经发生的行为里。

真正值得做的机会，通常不是“我能做什么”，而是：

> 哪里已经有人在痛、有人在搜、有人在骂、有人在付钱，但结果页还是垃圾？

那里就是鱼密度最高的地方。

---

## 核心公式

```text
鱼密度 = 需求增长 × 痛苦强度 × 付费能力 × 分发可达性 ÷ 竞争成熟度
```

一个机会是否值得做，不取决于它看起来多新、多酷，而取决于这几个问题：

- 需求是不是正在增长？
- 用户是不是足够痛？
- 用户有没有付费能力？
- 你能不能低成本触达这些用户？
- 现有竞争是否还很弱？
- 你能不能在 24 到 72 小时内做出一个比现有结果更有用的 MVP？

热词不是产品，热词只是入口。

真正的产品结构应该是：

```text
新词 / 热词 + 特定人群 + 具体动作 + 立刻省钱 / 省时间 / 避风险
```

例如：

- 不要做“AI Agent 工具”，而要做“客服团队的 AI Agent 测试日志分析器”。
- 不要做“MCP 工具”，而要做“MCP Server 配置检查器 for Claude / Cursor 用户”。
- 不要做“TikTok Trend 工具”，而要做“Shopify 卖家的 TikTok 热门话题脚本生成器”。

越具体，越接近钱。

---

## 最好的 8 类鱼池

### 1. 新词热词池

新游戏、新 AI 模型、新 App、新角色、新功能、新政策、新硬件、新梗、新协议、新平台变化，都可能产生短期机会。

判断标准：

- 有人开始搜索；
- 搜索量突然上升；
- SERP 结果页很空；
- 竞品页面很薄；
- 用户问题很具体；
- 你能在 24 小时内做出比前十结果更有用的页面或工具。

新词不是让你追热点，而是让你吃信息不对称。

适合做的不是普通文章，而是：

- 工具；
- 数据库；
- 生成器；
- 查询器；
- 检查器；
- 模板库；
- 目录页；
- 配置器。

重点不是“这个词火不火”，而是它背后有没有具体动作。

好动词包括：

```text
安装、配置、迁移、导出、修复、比较、生成、检查、监控、同步、部署、转换、恢复、合规、自动化、降成本
```

坏动词包括：

```text
了解、围观、吐槽、转发、收藏、感觉不错
```

如果用户只是问“这是什么”，可能只是内容机会。

如果用户在问“怎么配置”“有没有模板”“为什么报错”“有没有替代品”“能不能自动化”，这才更接近付费机会。

---

### 2. 抱怨池

不要只看用户说“我想要什么”，要看用户在骂什么。

去这些地方挖：

- Reddit；
- X / Twitter；
- Discord；
- GitHub Issues；
- App Store 差评；
- Chrome 插件差评；
- Shopify App 差评；
- G2 / Capterra 差评；
- 小红书评论区；
- YouTube 教程评论区。

重点看二星、三星、长评论、反复出现的问题。

典型高价值抱怨：

```text
太贵
太慢
不会用
没有中文
没有批量处理
导出有水印
登录太麻烦
不支持 Shopify
不支持 CSV
移动端很烂
客服不回复
配置太麻烦
同步不了
导出不了
每次都要手动
有没有替代品
```

抱怨就是未被满足的付费需求。

尤其是成熟产品的重复差评，非常值钱。因为成熟产品已经证明了市场存在，而差评说明供给仍然没有满足用户。

你的机会不是重新做一个巨型竞品，而是做一个更窄、更便宜、更顺手的小功能。

---

### 3. 迁移池

一个需求在 A 市场已经赚钱，但在 B 市场还没人做，这就是迁移机会。

常见迁移方式：

- 美国有，中文没有；
- 英文 SEO 有，日语 / 韩语没有；
- Shopify 有，Temu 卖家没有；
- 开发者圈有，传统行业没有；
- 大企业有，小商家没有轻量版；
- 房地产摄影有，婚礼摄影没有；
- AI 视频工具有，但 YouTube 长视频封面批量化没有。

独立开发最容易发财的不是原创，而是迁移。

不要总想着发明电灯泡。普通人更现实的机会，是把电灯泡搬到还在点蜡烛的村子里。

但迁移不是简单翻译，而是重新适配：

- 支付方式；
- 语言习惯；
- 行业术语；
- 获客渠道；
- 合规要求；
- 使用场景；
- 价格水平；
- 用户真实工作流。

---

### 4. 工具替代池

用户现在用 Excel、Notion、Google Sheet、手动复制、截图、临时 Prompt、人工外包解决的问题，都可能变成小工具。

判断句式：

```text
用户现在为了完成 X，需要打开 5 个页面，复制 3 次，手动整理 20 分钟。
```

这就是产品机会。

你的目标不是一开始做 SaaS，而是先问：

> 能不能把 20 分钟变成 20 秒？

典型机会：

- Amazon Listing 批量检查；
- Shopify 商品图生成；
- Google Trends 批量分析；
- Reddit 抱怨抓取；
- 城市服务商数据库；
- AI Prompt 模板生成器；
- SEO 页面标题批量生成；
- YouTube 视频脚本拆解；
- CSV 转换；
- 批量导出；
- 批量截图；
- 自动报告生成。

很多赚钱机会都藏在“脏、碎、烦”的重复劳动里。

---

### 5. 监管 / 平台变化池

平台规则一变，就会产生新需求。

变化来源包括：

- Google 改 SEO；
- YouTube 改规则；
- Stripe 改风控；
- Apple 改审核；
- TikTok 改推荐；
- Shopify 出新接口；
- AI 平台发布新模型；
- API 收费方式变化；
- 插件系统上线；
- 某类合规要求变化。

这种机会很肥，因为用户不是好奇，而是恐慌。

恐慌型需求比兴趣型需求更值钱。

用户会搜索：

```text
怎么迁移
怎么兼容
怎么配置
怎么避免封号
怎么符合新规则
怎么恢复旧功能
怎么替代某个工具
```

适合做：

- 迁移工具；
- 兼容性检查器；
- 政策解释器；
- 配置生成器；
- 监控报警；
- 新规则 Checklist；
- 一键修复工具。

---

### 6. 排行榜池

排行榜不是用来崇拜的，而是用来拆解的。

可以看：

- Product Hunt；
- Toolify；
- TAAFT；
- App Store 榜单；
- Chrome Web Store；
- Shopify App Store；
- GitHub Trending；
- Hacker News；
- Reddit 热帖。

你要观察的不是“谁火”，而是：

- 新上榜产品解决了什么问题；
- 有没有低配版可以做；
- 有没有垂直行业版可以做；
- 有没有多语言版可以做；
- 有没有 SEO 词可以截流；
- 用户评论暴露了什么缺陷；
- 流量从哪里来；
- 钱从哪里来；
- 弱点在哪里。

不要拜榜单，要像屠夫拆牛一样拆榜单。

### 7. 低端服务池

很多独立开发者忽视传统行业，但这里往往有钱。

典型人群：

- 水管工；
- 屋顶维修；
- 婚礼摄影；
- 宠物美容；
- 牙医；
- 律师；
- 房产经纪人；
- 家庭清洁；
- 装修工；
- 陪产师；
- 健身教练；
- 保险代理；
- 招聘顾问；
- 会计师；
- 餐厅老板；
- 本地服务商。

他们不一定懂 AI，也不关心自动化概念，但他们懂一个词：

> 客户。

他们不是买技术，是买更多客户。

可以给他们做：

- 线索提醒；
- 报价生成；
- 评论管理；
- 照片优化；
- 本地 SEO 页面；
- 自动回复；
- Google Business Profile 优化；
- 广告素材生成；
- 预约表单；
- 竞争对手监控；
- 开放日跟进短信；
- 客户回访邮件；
- 社媒内容生成。

命名越具体越好。

不要叫：

```text
AI Automation Platform
```

要叫：

```text
房产经纪人开放日跟进短信生成器
```

名字越土，钱味越重。

### 8. 付费验证池

最好的需求验证，是别人已经收过钱。

看哪里已经有人花钱：

- Google Ads 投放页面；
- App Store 内购；
- Chrome 插件高级版；
- Shopify App 收费；
- Gumroad 模板销量；
- Etsy 数字产品销量；
- Udemy 课程销量；
- Fiverr / Upwork 高频服务；
- Reddit 上有人问愿不愿意付钱；
- 竞品 Pricing 页面；
- 竞品 Alternative 页面。

别人已经收钱的地方，就是鱼已经咬钩的地方。

你的任务不是证明需求存在，而是证明：

> 你能不能用更小、更快、更垂直的方式切进去。

## 新词热词的主动测试流程

### 第一步：每天采 100 个信号

来源包括：

- Google Trends；
- TikTok Creative Center；
- Reddit；
- Product Hunt；
- Hacker News；
- GitHub Trending；
- 官方产品更新；
- API 文档；
- 开发者社区；
- App Store；
- Chrome Web Store；
- Shopify App Store；
- YouTube 评论区；
- X / Twitter。

采集时先不要判断，不要兴奋，不要幻想。

只记录四件事：

```text
这个词是什么？
谁在讨论？
讨论里出现了什么动词？
有没有人因为它卡住？
```

名词是热闹，动词才是钱。

---

### 第二步：把热词变成搜索组合

不要只搜一个词，要搜组合词。

假设新词叫 X，马上测试：

```text
X template
X generator
X checker
X alternative
X API
X Chrome extension
X Notion template
X spreadsheet
X pricing
X open source
X error
X config
X export
X import
X automation
X for Shopify sellers
X for teachers
X for real estate agents
```

中文同理：

```text
X 模板
X 生成器
X 检查器
X 替代品
X 怎么用
X 配置
X 报错
X 自动化
X 插件
X 表格
X 教程
X 案例
```

关键不是搜索量一定要大，而是看供给质量。

如果搜索结果第一页全是大公司、成熟 SaaS、广告、评测站，说明你可能晚了。

如果第一页是论坛帖子、GitHub Issue、官方文档、零散教程、Reddit 问答，而且没有一个顺手工具，说明可能有机会。

---

### 第三步：筛选 10 个候选机会

从 100 个信号里，只留下 10 个。

筛选标准：

- 搜索量在涨；
- 竞争页面弱；
- 用户问题明确；
- 能做工具，而不是只能写文章；
- 24 到 72 小时能上线 MVP；
- 有英文长尾词；
- 有付费影子；
- 不明显侵权；
- 能做 30 到 300 个 pSEO 页面；
- 能叠加 AI / API / 数据库。

不要在第 23 个烂想法上谈理想。

理想很贵，早期更重要的是现金流。

---

### 第四步：给每个机会打分

满分 50 分：

| 指标 | 分值 | 高分信号 | 低分信号 |
|---|---:|---|---|
| 用户痛苦强度 | 10 | 报错、迁移、浪费时间、影响收入 | 只是好奇 |
| 搜索增长 | 10 | Rising、Breakout、社区讨论变多 | 只是媒体热闹 |
| 竞争弱度 | 10 | 只有教程、论坛、开源项目 | 已有一堆成熟 SaaS |
| 变现清晰度 | 10 | 开发者、商家、团队、专业人士愿意付费 | 纯娱乐用户 |
| 快速交付能力 | 10 | 1 到 3 天能做出小工具 | 需要半年研发 |

决策标准：

```text
低于 35 分：不做
超过 40 分：立刻做测试页
超过 45 分：注册域名，上工具，上追踪，上收款
```

## 先做诱饵，不做完整产品

没有验证前，产品越完整，死得越体面。

早期不要先做完整产品，而是做诱饵。

诱饵可以是：

- 一个 Landing Page；
- 一个免费小工具；
- 一个 Google Sheet 模板；
- 一个 Notion 模板；
- 一个 Chrome 插件假门；
- 一个 GitHub Repo；
- 一个 Loom 演示视频；
- 一个 Reddit 帖子；
- 一个 X / Twitter Demo；
- 一个 Gumroad 预售页；
- 一个 9 美元 Refundable Deposit；
- 一个“我帮你手动做一次”的服务页。

你不是在骗人，而是在验证用户是不是真的愿意动一下手指，甚至掏钱。

一个诱饵页面只需要五个东西：

```text
标题：直接命中人群和动作
痛点：说明用户现在为什么痛
演示：截图、GIF、视频都可以
价格：哪怕是 9 美元、19 美元、49 美元
按钮：Join waitlist / Buy now / Get beta / Book setup
```

好标题示例：

```text
MCP Config Checker for Claude Desktop
```

一句话描述：

```text
Paste your MCP config and find broken servers, missing permissions, and unsafe commands in 30 seconds.
```

这比“下一代 AI 基础设施平台”强得多。

前者像能收钱的东西，后者像创业比赛 PPT。

## 48 小时验证法

不要先做完整产品。

48 小时内只做：

- 一个首页；
- 一个真实可用的小工具；
- 一个 Pricing 假门或 Waitlist；
- 10 个长尾页面；
- Plausible / PostHog 埋点；
- 一个反馈入口；
- 一个邮件收集；
- 一个免费额度。

看三个指标：

```text
有没有自然搜索曝光？
有没有用户真的使用工具？
有没有人点 Pricing / Export / Download / Unlock？
```

如果有人点价格、导出、下载、解锁，说明已经接近付费意图。

## 主动投放鱼钩

早期不要等 SEO，要主动测试。

每个想法至少投 5 个地方：

- 相关 Reddit 社区；
- 相关 Discord / Slack / Telegram 群；
- GitHub Issue 或 Discussion；
- Hacker News / Show HN / 产品社区；
- X / Twitter 小号 Demo；
- 小红书 / 抖音 / TikTok；
- 私信 10 个目标用户；
- 发给 5 个正在做类似事情的人。

话术不要像广告，要像一个刚解决自己问题的人。

好话术：

```text
我最近在配 MCP server，经常因为一个小 JSON 错误浪费半小时，所以做了个小检查器。把 config 贴进去，它会告诉你哪里坏了。有人也遇到这个问题吗？
```

坏话术：

```text
请使用我们革命性的 AI 开发者效率平台。
```

人话永远比广告话值钱。

## 7 天决策法

7 天内做判断，不要长期幻想。

继续加码的信号：

- 有搜索曝光；
- 有真实使用；
- 有收藏；
- 有询问；
- 有人问价格；
- 有人问权限；
- 有人问集成方式；
- 有人问能不能支持他的场景；
- 有人愿意预付费；
- 有人愿意把真实数据交给你试。

停止或调整的信号：

```text
只有访问没有使用：页面骗到了点击，但产品不对。
有使用没付费：价值不够强，或付费点错。
连曝光都没有：词错了，或竞争判断错了。
只有点赞收藏，没有注册、私信、付款：弱信号。
```

判断标准：

```text
50 个精准访问里，有 5 个留下邮箱，可以继续。
50 个精准访问里，有 1 个付费或预付，可以继续。
10 个目标用户私信里，有 3 个认真回复痛点，可以继续。
```

用户说“这个很有意思”，通常没钱。

用户说“这个能不能明天给我用，我现在就卡在这里”，钱味就出来了。

## 先人工交付，再自动化

独立开发者不要一开始追求系统完美。

早期可以前台像软件，后台像人肉。

例如：

- 用户上传文件，你手动处理；
- 用户提交配置，你用脚本和 AI 检查；
- 用户要报告，你先人工生成 PDF；
- 用户要自动化，你先帮他跑一次；
- 用户要 SEO 页面，你先手动交付第一批。

有 5 个付费用户之后，再自动化。

没付费前，自动化大概率是在给空气装修豪宅。

## 热词产品化的 6 个方向

热词本身一般赚不到钱，围绕热词的配套工具更容易赚钱。

### 1. 解释类

适合引流和 SEO。

```text
What is X
X 中文指南
X for beginners
X examples
X best practices
```

### 2. 模板类

适合卖 9 到 49 美元的小产品。

```text
X prompt template
X config template
X workflow template
X policy template
X agent skill template
```

### 3. 检查类

更容易变成 SaaS，因为检查、监控、报警有持续价值。

```text
X config checker
X security scanner
X compatibility checker
X compliance checker
X migration checker
```

### 4. 目录类

适合新生态早期。

```text
Best X tools
X server directory
X plugin marketplace
X examples gallery
X use case library
```

### 5. 迁移类

迁移是强痛点，通常更接近付费。

```text
Move from A to B
Convert A to B
Export from A
Import into B
A alternative for B users
```

### 6. 垂直行业类

泛词竞争大，垂直词竞争小。

```text
X for lawyers
X for dentists
X for Shopify sellers
X for teachers
X for real estate agents
X for indie hackers
```

真正的机会不是“AI Tool”，而是：

```text
AI invoice dispute assistant for Shopify merchants
```

越窄，越锋利。

---

## 其他高鱼密度方法

### 1. 大工具拆小功能

很多大 SaaS 越做越臃肿，价格越来越贵。

你可以找到其中用户最常用、最痛的一个小功能，单独拆出来做。

搜索方式：

```text
[大工具名] alternative
[大工具名] too expensive
[大工具名] export
[大工具名] missing feature
[大工具名] for small business
[大工具名] pricing
```

如果一个大工具每月 99 美元，但用户其实只想要里面的自动导出、报表、截图、提醒、批量处理，你可以做一个 19 美元的小工具。

这叫从大象身上割肉，不叫重新发明大象。

---

### 2. 淘金热卖铲子

当一个新生态起来，不一定要自己下场淘金，可以给淘金者卖铲子。

例如：

- AI Agent 热：做 Agent 测试、日志、权限、监控、安全；
- MCP 热：做 MCP 目录、配置器、检查器、安全扫描、部署模板；
- Claude Skills 热：做 Skill Builder、Skill Marketplace、Skill Validator、行业 Skill 包；
- TikTok Shop 热：做选品提醒、脚本生成、达人匹配、评论分析；
- 创作者经济热：做脚本、素材、选题、自动剪辑、数据面板。

淘金的人会死一批，但卖铲子的人只要工具有用，就有机会活下来。

---

### 3. 开源项目商业化

GitHub Trending 不是用来膜拜大神，而是看哪个开源项目正在被大量采用。

一旦开发者采用，马上会出现非技术用户不会用的问题。

机会包括：

- 一键部署；
- 云托管版本；
- 图形界面；
- 企业权限；
- 监控日志；
- 团队协作；
- 模板市场；
- 付费支持；
- 安全加固；
- 文档翻译。

不要跟开源项目竞争，而是帮它变得能被普通人使用。

很多钱不是来自代码本身，而是来自降低使用门槛。

---

### 4. 服务先行，再软件化

很多用户不想买工具，他们想买结果。

他们不想买 SEO 工具，他们想要 100 篇能带流量的页面。

他们不想买视频工具，他们想要 30 条能发的短视频。

他们不想买数据分析工具，他们想知道哪个产品该上架。

所以可以先人工交付，收钱，然后把重复部分自动化。

适合服务先行的方向：

- 帮 Shopify 商家每周生成 TikTok 脚本；
- 帮独立开发者做竞品监控；
- 帮房产经纪人生成本地 SEO 页面；
- 帮小企业整理 Google Review 回复；
- 帮开发者检查 MCP / Agent 配置；
- 帮招聘团队筛简历并生成面试问题。

先卖结果，别急着卖软件。

软件是交付系统，不是信仰。

---

### 5. 连接缝隙

很多钱藏在两个工具之间。

典型痛点：

```text
A 工具的数据导不进 B
B 的报表没法给老板看
C 的截图没法自动整理
D 的客户消息不能同步到 CRM
E 的 AI 输出不能进入工作流
```

关键词：

```text
sync
export
import
webhook
Zapier alternative
connector
integration
API wrapper
bulk upload
CSV to X
X to PDF
X to Notion
X to Airtable
X to Slack
```

不要小看 CSV。

这个世界一半的商业自动化，底层都是一堆人在痛苦地搬 CSV。

---

### 6. 模板、计算器、检查清单

不要嫌模板、计算器、检查清单低级。

它们低成本、低风险、适合 SEO、适合快速上线，也适合后续组合成工具库。

可以做：

- ROI Calculator；
- Pricing Calculator；
- Invoice Generator；
- Contract Checklist；
- Ad Script Generator；
- TikTok Hook Generator；
- SEO Brief Generator；
- Compliance Checklist；
- Grant Eligibility Checker；
- Website Audit Tool；
- Resume Scanner；
- Landing Page Grader；
- YouTube Title Generator；
- Shopify Product Description Generator。

一个工具每月只赚 100 美元也没关系。

100 个工具就是另一回事。

---

### 7. 数据监控和提醒

凡是信息会变化，而且变化会影响钱，就有机会。

可监控对象：

- 价格变化；
- 库存变化；
- 政策变化；
- 竞品变化；
- 招聘变化；
- 域名变化；
- 排名变化；
- 评论变化；
- 广告变化；
- GitHub Release 变化；
- API 文档变化；
- App Store 评价变化。

产品形态很简单：

```text
每天监控，变化时提醒。
```

提醒类产品容易订阅化，因为它不是一次性工具，而是持续帮用户避免损失或抓住机会。

---

### 8. 竞品替代页

当一个产品已经证明市场存在，你可以做它的窄版替代品。

搜索：

```text
[竞品] alternative
[竞品] cheaper alternative
[竞品] for solo founders
[竞品] for agencies
[竞品] open source
[竞品] pricing
[竞品] vs [竞品]
```

页面结构：

```text
Best [竞品] alternative for [具体人群]
```

但不能只是写文章。

你要真的给一个小工具、模板、试用版，让用户马上体验。

这是 SEO + 产品的打法。

## 每天 2 小时执行 SOP

每天固定 2 小时捕鱼，不靠灵感，靠流程。

### 前 30 分钟：采词

看：

- Google Trends；
- TikTok Creative Center；
- Reddit；
- Product Hunt；
- Hacker News；
- GitHub；
- 官方产品更新；
- API 文档；
- YouTube 评论区；
- App Store / Chrome Store 差评。

只记词，不判断。

目标：每天发现 20 到 100 个信号。

---

### 第 2 个 30 分钟：找痛点

围绕每个词搜索：

```text
template
generator
checker
alternative
error
config
export
automation
for 某职业
```

看有没有人在问、骂、求工具。

重点找具体动作，不找泛泛兴趣。

---

### 第 3 个 30 分钟：看竞争

检查：

- Google 搜索第一页；
- Reddit 讨论；
- GitHub Issue；
- Product Hunt 同类产品；
- Chrome Web Store / App Store 差评；
- Shopify App Store 评论。

判断市场是否成熟。

如果全是巨头和成熟 SaaS，先跳过。

如果全是论坛、文档、教程、零散脚本，留下。

---

### 最后 30 分钟：撒一个钩子

每天至少做一个动作：

- 发一个帖子；
- 做一个落地页；
- 发一个 Demo；
- 私信 10 个人；
- 写一个小工具页面；
- 做一个 Gumroad 预售；
- 丢一个 GitHub Repo；
- 做一个 Waitlist；
- 上一个 Pricing 假门。

每天至少一个钩子。

不要空想。

---

## 一周产出标准

一周下来，理想状态应该有：

```text
100 个词
20 个候选机会
10 个诱饵
2 个小 Demo
1 个最有付费信号的方向
```

这才叫主动测试。

不是做一个大产品赌半年，而是每周不断下钩、杀掉烂想法、留下有钱味的方向。

---

## 最适合独立开发者的优先级

早期不要做半年才能上线的大东西。

优先做这四类：

### 1. 热词周边小工具

围绕新协议、新平台、新 AI 工作流，做：

- 配置器；
- 检查器；
- 模板；
- 目录；
- 安全扫描；
- 部署工具。

### 2. 差评挖出来的窄功能

找成熟工具的二三星评价，做一个更窄、更便宜、更顺手的替代功能。

### 3. 服务先行

先卖结果，人工交付，再把重复部分自动化。

特别适合不会代码但执行力强的人。

### 4. 模板 / 计算器 / 检查器工具库

低成本、低风险、可积累 SEO、可复用组件。

不要嫌小。

现金流很多时候就是从小洞里先流出来的。

## 日常检查清单

每天问自己：

- 今天有没有发现 20 个新词？
- 有没有看 50 条用户抱怨？
- 有没有拆 5 个新产品？
- 有没有找到 3 个弱 SERP？
- 有没有上线 1 个测试页？
- 有没有记录搜索曝光？
- 有没有加一个真实工具？
- 有没有主动发给目标用户？
- 有没有看到使用、注册、询问、付费信号？
- 这个页面是否比 Google 前十更有用？

如果没有，就不是在捕鱼，只是在刷信息流。

---

## 最核心的判断问题

不要问：

```text
我能做什么产品？
```

要问：

```text
谁现在很痛？
他在哪里抱怨？
他已经用什么烂办法解决？
我能不能三天做一个更省事的版本？
他愿不愿意今天付 9 美元、29 美元、99 美元？
```

能回答这五个问题，鱼就不是鱼了，是钱在水里游。

---

## 一句话版本

独立开发不是浪漫地做产品，而是冷酷地寻找用户已经表现出来的欲望。

不要去海里撒网。

去水面已经冒泡的地方下钩。
```
